# 月次P&L変動分析スキル

## 概要

月次P&Lデータ（xlsx）を入力とし、社外役員・監査役向けの変動分析を3段階で実施する。
「何が動いたか」→「どこで動いたか」→「なぜ動いたか」を構造的に明らかにする。

## 分析フレームワーク（3段階）

### Stage 1: 全体変動の把握

```
## 分析対象

全体の売上に対して、以下のコスト区分ごとに増減を分析してください。

[売上連動コスト]（真の変動費）
  - {費目1}
  - {費目2}

[裁量的コスト]
  - {費目3}

## 分析内容

### (1) 全体推移（対象期間を全月出力）
- 売上高・各コスト項目の月次絶対値
- 各項目の対売上比率の推移
- 前月比（金額・%）

### (2) 因果関係の検証
- 売上連動コストは売上に比例しているか？（相関係数で検証）
- 裁量的コストは売上に比例しているか？（相関係数で検証）
- コスト1円あたり売上の推移（効率の変化）
- コスト増加額の要因分解（各費目の寄与率）
```

**Stage 1のチェックポイント:**
- [ ] 売上連動コストの対売上比率は安定しているか（±2pt以内が正常）
- [ ] 裁量的コストの対売上比率は拡大していないか
- [ ] コスト増加の寄与率で最大要因を特定したか

### Stage 2: セグメント別の傾向判定

```
### (3) セグメント別に同じ傾向か？
- 全セグメントでStage 1の(1)(2)を実施
- コスト/売上比率が悪化しているセグメントを特定
- 悪化が「全社共通」か「特定セグメントに集中」か分類

## 分類基準
- "同傾向": コスト増 > 売上増（全体と同じ悪化パターン）
- "効率的": 売上増 >= コスト増
- "コストなし": 対象コストがゼロまたは微小
- "両方減": コストも売上も減少
- "コスト増・売上減": 最非効率パターン

## 集中度分析
- 問題セグメントが全コスト増加額の何%を占めるか
- 問題セグメント vs 残りセグメントの比率比較
```

**Stage 2のチェックポイント:**
- [ ] 全セグメントを5分類に分けたか
- [ ] 集中度（上位N社で全体の何%か）を算出したか
- [ ] 問題セグメント群 vs 残りの比較テーブルを作成したか

### Stage 3: 時系列変化と要因分解

```
### (4) 時系列の構造分析
- 悪化がいつ始まったか（転換点の特定）
- 前半 vs 後半の比較
- 比率変動の要因分解:
  - 売上要因: コストが一定なら比率はどう変わったか
  - コスト要因: 売上が一定なら比率はどう変わったか
  - 構成効果: セグメントミックスの変化による影響
- 月別の最大寄与セグメントを特定
```

**Stage 3のチェックポイント:**
- [ ] 転換点（急変した月）を特定したか
- [ ] 前半/後半の比較テーブルを作成したか
- [ ] 比率変動を「売上要因」「コスト要因」「構成効果」に分解したか
- [ ] 月別の最大寄与セグメントを特定したか

## 出力形式

### 必須テーブル
1. **月次推移テーブル**: 全月×全項目の絶対値
2. **対売上比率テーブル**: 全月×全項目の比率
3. **要因分解テーブル**: 寄与度（pt, %）
4. **セグメント分類テーブル**: 5分類×該当セグメント
5. **前半vs後半比較テーブル**: 平均値比較
6. **結論コメント**: 各Stage末に日本語で1-2文

### 数値フォーマット
- 金額: 千円単位（カンマ区切り）
- 比率: 小数第1位まで（%）
- 変化: +/-符号付き
- 相関係数: 小数第2位まで

## プロンプトテンプレート（コピペ用）

```
全体の売上増に対して、以下のコスト区分ごとに増減を分析してください。

[売上連動コスト]（真の変動費）
  - {費目を列挙}

[裁量的コスト]
  - {費目を列挙}

## 分析内容

### (1) 全体推移（全月を出力）
- 月次絶対値、対売上比率、前月比（金額・%）

### (2) 因果関係の検証
- 相関係数、コスト効率の推移、増加額の要因分解（寄与率）

### (3) セグメント別に同じ傾向か？
- 全セグメントで(1)(2)を実施
- 悪化が全社共通か特定セグメントに集中か分類
- 問題セグメント群 vs 残りの比較

### (4) 時系列の構造分析
- 転換点の特定、前半vs後半比較
- 比率変動の要因分解（売上要因 vs コスト要因 vs 構成効果）
- 月別の最大寄与セグメントを特定

## 出力
- 全て月次テーブルで出力
- 各分析に結論コメントを付ける
```

## プロンプト設計の3原則

### 1. Yes/No質問を避け「分解」を求める
- NG: 「増えていますか？」
- OK: 「増加額を要因別に分解し、寄与率を算出してください」

### 2. 比較には「分類基準」を含める
- NG: 「同じ傾向ですか？」
- OK: 「全社共通か特定セグメントに集中か、5分類で判定してください」

### 3. 時系列には「転換点」と「前後比較」を求める
- NG: 「推移を見せて」
- OK: 「転換点を特定し、前半vs後半で比較してください」

## 適用事例

### デジコン事業の変動費分析（2025-2026）
- Stage 1: 変動費+26%の増加は100%広告宣伝費が原因（真の変動費は-9.8%）
- Stage 2: 問題は3セグメント（はやとも系+自社メディア）に集中（全広告費増の118%）
- Stage 3: 転換点は25年11月。直近3ヶ月で悪化の91%が発生。要因は100%広告費増（売上はむしろ+9.6%成長）

## 注意事項

- データの列数・行数が月ごとに異なる場合がある（セグメント追加、人員変動等）
- 行のマッチングはラベルテキストで行い、行番号に依存しない
- 合計列は常に最終列を使用
- 相関係数はデータ数が少ない場合（<6）は参考値として扱う
